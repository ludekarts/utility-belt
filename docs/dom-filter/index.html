domFilter(config)

Generates filtering function for sets of DOM elements. Can be quick and dirty way to provide filtering functionality for pages with any repetetive content.

Usage

  import { domFilter } from "utility-belt";
  ...

  const \\filter\\ = domFilter({
    selector: "li.list-item",
  });

  ...

  input.addEventListener("input"), event => {
    \\filter\\(event.target.value);
  });

Props (config)

Name Type Description
selector String CSS selector for nodes to be added to filtering list
extractContent Function Provides alternative way for selecting item content for filtering purposes. Fn takes two arguments:
  • item - reference to currently prcess list element
  • index - current item index
uiRender Function Provides a custom render function for the element. Fn takes three arguments:
  • entry - current entry { text: "item text" ref: "element reference" }
  • phrase - current phrase
  • store - mutable object that is preserved though entire list processing. Can be used as cache or temporary storage
matchFn Function Custom matching function. Allow you to provide your own matching algorithm. Fn takes two arguments:
  • text - entry text value
  • phrase - current phrase
matchIfEmptyPhrase Boolean Default true. Display list items if phrase is empty
